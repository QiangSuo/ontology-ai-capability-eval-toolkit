# Human Effort Metrics

## 1. 目标

人机协同成本用于衡量 AI 工具组合是否真正降低本体项目交付成本。一个工具即使输出质量尚可，如果需要大量人工分片、修复、重写和专家判断，也不适合作为主力工具。

## 2. 记录粒度

必须按任务记录，并在评估结束后汇总。

每个任务至少记录：

- 任务编号。
- 测试模式：fixed 或 free。
- 开始时间和结束时间。
- 操作员输入次数。
- 模型追问次数。
- 人工 prompt 改写次数。
- 文件分片次数。
- JSON 修复次数。
- 失败重试次数。
- 专家介入次数。
- 人工修改输出次数。
- 人工重写比例。
- 最终可用输出比例。
- 操作员备注。

## 3. 指标定义

### 3.1 总耗时

从操作员开始执行任务到保存该任务最终输出的时间。

记录方式：

- `< 15 分钟`
- `15-30 分钟`
- `30-60 分钟`
- `1-2 小时`
- `> 2 小时`

### 3.2 操作员输入次数

操作员向工具输入 prompt、文件内容、修复要求或补充说明的总次数。

不计入：

- 单纯复制工具输出保存到文件。
- 打开文件查看但未输入给工具。

### 3.3 模型追问次数

模型要求操作员补充信息、确认范围或重新提供材料的次数。

区分：

- 合理追问：因权限或材料缺失导致。
- 不合理追问：prompt 已明确但模型仍无法执行。

### 3.4 Prompt 改写次数

操作员为了让工具完成任务而改写或强化 prompt 的次数。

固定模式中，prompt 改写原则上不允许；若发生，必须标记为协议偏离。

### 3.5 文件分片次数

因上下文限制、文件读取限制或工具能力限制，操作员手工拆分输入的次数。

### 3.6 JSON 修复次数

为获得可解析结构化输出而要求模型修复 JSON 的次数。

### 3.7 失败重试次数

任务因工具崩溃、输出无效、超时、格式错误或误解任务而重新执行的次数。

### 3.8 专家介入次数

FDE 核心团队或本体专家参与判断、修正、重写或指导的次数。

需要区分：

- 流程指导。
- 业务判断。
- 本体建模判断。
- 输出重写。

### 3.9 人工重写比例

最终可用输出中由人工重写或实质性补充的比例。

建议区间：

| 区间 | 含义 |
|---|---|
| 0-10% | AI 输出基本可用。 |
| 10-30% | 需要常规修订。 |
| 30-50% | 人工负担较高。 |
| >50% | 人工成为主要产出来源。 |

### 3.10 最终可用输出比例

AI 原始或轻度修复输出中，能进入本体草案评审的比例。

## 4. Human Effort Index

使用 0-5 记录人机协同成本。

| HEI | 名称 | 判定标准 | 交付含义 |
|---:|---|---|---|
| 0 | 几乎无需人工 | 任务一次完成，输出结构合规，人工只做保存和抽查。 | 可自动化或半自动化。 |
| 1 | 低干预 | 需要少量追问或格式修复，无业务重写。 | 可作为主力能力。 |
| 2 | 可接受干预 | 需要分片、补证据或 1-2 次修复，但人工不重写核心内容。 | 可进入 POC。 |
| 3 | 高干预 | 需要频繁 prompt 调整、人工判断和局部重写。 | 只能有限范围使用。 |
| 4 | 人工主导 | AI 输出只能作为参考，人工重写 30-50%。 | 不适合主流程。 |
| 5 | 不可交付 | 人工重写超过 50%，或专家成为主要产出来源。 | 不应作为交付工具。 |

## 5. HEI 判定规则

### HEI 0

同时满足：

- 任务一次完成。
- JSON 一次合规。
- 无专家介入。
- 人工重写比例 < 5%。
- 无红线。

### HEI 1

满足：

- 追问或修复次数 <= 1。
- 人工重写比例 < 10%。
- 无业务性重写。
- 输出可直接进入审核。

### HEI 2

满足：

- 追问或修复次数 <= 3。
- 人工重写比例 < 30%。
- 专家介入 <= 1 次，且只用于复核。
- 输出主体来自 AI。

### HEI 3

任一满足：

- 追问或修复次数 4-6 次。
- 人工重写比例 30-50%。
- 专家介入 2 次以上。
- 操作员需要频繁拆分输入或纠正方向。

### HEI 4

任一满足：

- AI 输出大部分需要人工重写。
- 专家参与成为任务完成必要条件。
- 固定模式无法完成，只能自由模式勉强完成。

### HEI 5

任一满足：

- 人工重写比例 > 50%。
- 输出不可复盘。
- 工具多次卡死或无法完成任务。
- 出现人工答案伪装为模型输出。

## 6. 与评分的关系

| HEI | 人机协同成本能力等级 | 总分影响 |
|---:|---|---|
| 0 | A | 无扣分。 |
| 1 | A | 无扣分。 |
| 2 | B | 可进入 POC。 |
| 3 | C | 总分建议最高 B-，需 FDE 看护。 |
| 4 | D | 总分最高 C+，不适合主流程。 |
| 5 | D | 触发红线或准入失败。 |

## 7. 固定模式与自由模式记录

固定模式：

- 记录真实执行成本。
- 不允许通过大幅改 prompt 降低成本指标。
- Prompt 改写视为协议偏离。

自由模式：

- 允许工具特性适配。
- 必须记录所有额外人工操作。
- 最终报告必须展示 free mode 分数提升是否依赖高人工成本。

## 8. 记录模板

每个任务建议记录如下字段：

```text
task_id:
mode:
operator_id:
start_time:
end_time:
elapsed_minutes:
operator_input_count:
model_followup_count:
prompt_rewrite_count:
manual_chunking_count:
json_repair_count:
retry_count:
expert_intervention_count:
manual_edit_count:
manual_rewrite_ratio:
usable_output_ratio:
human_effort_index:
notes:
```

## 9. 操作员填写要求

- 不要只记录成功步骤，失败和返工必须记录。
- 专家口头指导也要记录。
- 人工修复 JSON 与人工重写业务内容必须区分。
- 若无法估算人工重写比例，应保守上调 HEI。
- 若客户工具无法导出完整日志，操作员必须手工补充 execution log。
