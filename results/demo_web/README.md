# Demo Web Result

This directory demonstrates the optional C-stage Web snapshot extension from the evaluator side.

- Task 12 extracts Web-derived ontology candidates from offline HTML snapshots.
- Task 13 analyzes page-map navigation and business flows.
- Task 14 fuses MVP baseline context with Web-derived outputs.
- `scoring/` is generated by `scoring/score_auto.py` and includes optional Web evidence checks.

The demo uses `input_mode = html_snapshot`; it does not claim live Web crawling, login, JavaScript execution, browser automation or backend API behavior.

## Input Boundary

The simulated AI task inputs are limited to the prompt files and the optional Web input artifacts listed in each task result wrapper. Evaluator-only `gold/*`, `known_conflicts*`, scoring outputs, and existing demo outputs are not task inputs and must not be provided to an AI tool under evaluation.

The `web:*` evidence handles in normalized outputs are output-side traceability handles. They are checked against evaluator-only Web evidence maps during scoring, but those gold/reference files are not part of the task input bundle.
