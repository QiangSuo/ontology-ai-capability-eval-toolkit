# core

## 用途

`core/` 存放所有客户通用的核心方法论，包括能力模型、任务协议、评分总则、红线规则、降级规则和准入标准。

## 包含内容

后续可包含：

- 能力模型定义
- Smoke / Standard / Deep Agentic 任务协议
- 评分权重和红线规则
- 准入门槛
- 固定模式与自由模式规范
- 降级路径规则

## 输入

- FDE 本体项目交付方法论
- 客户现场常见权限限制
- 标准测试任务结果
- 自动评分和人工评分结果

## 输出

- 能力域定义
- 任务执行协议
- 评分和准入规则
- 报告生成所需的核心判断依据

## 使用规范

- `core/` 不应绑定某个具体客户。
- 客户特定内容应放入 `customer_profiles/` 或 `results/`。
- 行业特定数据应放入 `datasets/` 的行业扩展目录。
- 修改能力模型、红线或准入标准前，应由 FDE 负责人确认。
