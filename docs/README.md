# docs

## 用途

`docs/` 存放补充设计文档、术语表、架构说明、决策记录和团队协作规范。

## 包含内容

后续可包含：

- 术语表
- 项目结构说明
- ADR 架构决策记录
- 评估案例复盘
- 常见问题
- 贡献指南

## 输入

- implementation plan
- task breakdown
- 试运行反馈
- FDE 负责人确认的设计决策

## 输出

- 团队知识沉淀
- 设计决策记录
- 后续维护说明

## 使用规范

- docs 中的内容应补充核心文档，不替代核心规范。
- 重要设计变更应记录原因和影响。
- 与客户相关的复盘应先脱敏。
