# Roadmap

## MVP 阶段

目标：形成可执行、可复盘、可用于客户初筛的工具包。

范围：

- 一个通用业务系统数据集设计：企业采购审批与合同履约管理系统。
- 四类输入：源代码、DDL、样例数据、文档。
- Smoke Test 和 Standard Test。
- 基础能力模型、评分规则、报告模板和操作员手册。

主要交付：

- 项目基础目录和核心文档。
- 能力模型。
- 设计原则。
- 任务拆解。
- 后续 schema 设计说明。
- 后续评分规则说明。
- 后续操作员手册和报告模板。

不包含：

- 实际评分脚本。
- 实际 JSON Schema 文件。
- 实际测试数据集。
- 截图、Web 和行业增强包。
- Dashboard。

## V1 阶段

目标：支持更多真实客户现场场景。

新增能力：

- 截图输入和 UI 理解测试。
- Web 页面快照和页面地图测试。
- Level 3/4 混乱数据集。
- 能源、金融、制造、政务、供应链行业增强包。
- 工具画像和 benchmark history。
- 报告半自动生成。

成功标准：

- 能用于多个客户现场横向比较。
- 能记录不同工具组合的能力画像。
- 能明确工具适合和不适合的 POC 范围。

## V2 阶段

目标：从评估工具包升级为长期标准化能力平台。

新增能力：

- Deep Agentic Test。
- 变更影响分析测试。
- 跨客户对比 dashboard。
- 高级本体质量评估。
- 操作员培训与认证。

成功标准：

- 能长期沉淀跨客户、跨工具、跨模型能力趋势。
- 能支持 FDE 团队制定工具准入和 POC 范围策略。
- 能识别接近 Claude Code / Codex 自动化流水线能力的客户内网工具组合。

## 当前里程碑

当前里程碑只完成项目目录和文档骨架，为后续逐步实现打基础。
